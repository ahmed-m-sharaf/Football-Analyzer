# Football Analytics System

A modular AI-powered football analytics system that detects players, referees, and the ball, tracks objects across frames, assigns players to teams, computes match statistics, and generates an annotated output video.

# Folder Description

## `data/`

Stores all project resources including videos, trained models, cached files, and images.

### `data/videos/`

Contains football match videos.

- **input/** – Raw input videos.
- **output/** – Processed videos generated by the pipeline.

---

### `data/models/`

Stores all pretrained models used by the project.

Examples:

- Player Detection Model
- Ball Detection Model
- Field Keypoint Detector
- Team Classification Model

---

### `data/images/`

Contains images used during development or visualization.

Examples:

- Sample frames
- Football pitch templates
- Logos
- Testing images

---

### `data/cache/`

Stores temporary intermediate files to speed up processing.

Examples:

- Tracking results
- Homography matrices
- Detection cache
- Intermediate JSON files

This folder can be safely deleted and regenerated.

---

# `notebooks/`

Contains Jupyter notebooks used for research, experimentation, and testing.

Examples:

- Detection experiments
- Tracking experiments
- Speed calculation testing
- Model evaluation

Production code should not be placed here.

---

# `src/`

Contains the complete source code of the project.

---

## `src/detection/`

Responsible for detecting all objects in every frame.

### Responsibilities

- Detect Players
- Detect Ball
- Detect Referees
- Apply post-processing
- Return bounding boxes and confidence scores

### Input

```
Video Frame
```

### Output

```
Players
Ball
Referees
Bounding Boxes
Confidence Scores
```

---

## `src/tracking/`

Assigns a unique ID to every detected object and tracks it across video frames.

### Responsibilities

- Multi-object tracking
- Player ID assignment
- Ball tracking
- Track interpolation
- Recover lost tracks

### Input

```
Detections
```

### Output

```
Tracked Players
Tracked Ball
Persistent IDs
```

---

## `src/field/`

Handles football field understanding and camera geometry.

### Responsibilities

- Detect field keypoints
- Estimate homography matrix
- Perspective transformation
- Convert image coordinates to real-world pitch coordinates

### Input

```
Video Frame
```

### Output

```
Pitch Coordinates
```

---

## `src/teams/`

Assigns each tracked player to the correct team.

### Responsibilities

- Jersey extraction
- Jersey color analysis
- Team clustering
- Goalkeeper assignment

### Input

```
Tracked Players
```

### Output

```
Player ID → Team ID
```

---

## `src/analytics/`

Computes player and team statistics.

### Player Statistics

- Speed
- Distance Covered
- Acceleration
- Sprint Detection

### Ball Statistics

- Ball Possession
- Ball Owner
- Number of Touches

### Team Statistics

- Team Possession
- Average Speed
- Total Distance Covered

### Input

```
Tracked Players
Ball
Pitch Coordinates
```

### Output

```
Player Statistics
Team Statistics
```

---

## `src/visualization/`

Draws all analytics on top of the video.

### Responsibilities

- Bounding Boxes
- Player IDs
- Team Colors
- Ball Path
- Speed Labels
- Distance Labels
- Possession Information
- Mini Pitch
- Statistics Overlay

### Output

```
Annotated Video
```

---

## `src/pipeline/`

Coordinates the execution of the complete football analytics pipeline.

### Responsibilities

- Execute modules in order
- Pass data between stages
- Manage the processing workflow
- Handle configuration

Pipeline Flow

```
Input Video
      │
      ▼
Detection
      │
      ▼
Tracking
      │
      ▼
Field Mapping
      │
      ▼
Team Assignment
      │
      ▼
Analytics
      │
      ▼
Visualization
      │
      ▼
Output Video
```

---

## `src/utils/`

Contains helper functions shared across multiple modules.

Examples:

- Geometry utilities
- Image processing
- Mathematical functions
- Video utilities
- Logging
- Configuration loading
- File handling

---

## `src/main.py`

Application entry point.

Responsibilities:

- Load project configuration
- Initialize all models
- Execute the complete pipeline
- Save generated outputs

Run the project:

```bash
python src/main.py
```
